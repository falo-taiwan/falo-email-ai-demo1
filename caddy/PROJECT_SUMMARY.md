## 1. System Architecture & Port Mapping
The gateway splits control, routing, and tunneling into three decoupled layers:

```
[ Public Web Traffic (HTTPS) ]
            │
            ▼
┌────────────────────────┐
│   Cloudflare Tunnel    │ (falo-dev-tunnel)
└───────────┬────────────┘
            │
      ┌─────┴────────────────────────────────┐
      ▼ (Port 8080)                          ▼ (Port 8088)
┌────────────────────────┐             ┌────────────────────────┐
│     Caddy Gateway      │             │    Python Admin UI     │
│ (Dynamic Redirections) │             │ (Auth: falo / force)   │
└────────────────────────┘             └────────────────────────┘
```

| Component | Public Domain | Local Host/Port | Protocol | Main Process / Executable |
| :--- | :--- | :--- | :--- | :--- |
| **Caddy Gateway** | `go.formosa-ai.com` | `localhost:8080` | HTTP | `caddy/bin/caddy` (daemon) |
| **Python Admin UI** | `go-admin.formosa-ai.com` | `localhost:8088` | HTTP | Flask 3.x (`caddy/admin_ui/app.py`) |
| **Caddy Admin API** | - | `localhost:2019` | HTTP | Embedded in Caddy |
| **Cloudflare Agent**| - | Managed | QUIC | `bin/cloudflared` (daemon) |

## 3. Core Operation Flows

### 3.1 Initializing and Compiling Configs
On startup, `app.py` executes `initialize_system()`, which:
1. Loads rules from `caddy/rules.json`.
2. Generates Caddyfile using `caddy_manager.py`.
3. Overwrites `caddy/Caddyfile`.
4. Boots up Caddy using `caddy start --config Caddyfile`.
5. Boots up `cloudflared` using configured Tunnel token in `.env`.

### 3.2 Rule Insertion & Zero-Downtime Hot-Reload
When rules are updated via the UI:
1. Rules array is saved back to `caddy/rules.json`.
2. `caddy_manager.py` compiles the new rules into `caddy/Caddyfile`.
3. Calls Caddy API reload: `caddy reload --config Caddyfile` (or makes a POST request to `http://localhost:2019/load`). This does NOT interrupt existing active TCP connections.

### 3.3 Rule Compiler Syntax Map
Rules in `rules.json` are parsed according to their `type`:

#### Redirect (HTTP 302 Jump)
```caddy
# [Description]
redir /path target_url 302
```

#### Proxy (Reverse Proxy)
Strips the prefix and forwards all subpaths to target:
```caddy
# [Description]
handle_path /path/* {
    reverse_proxy target_host_port
}
```

#### Static (Static File Directory Server)
Maps prefix to a local physical directory. Automatically appends trailing slashes using 308 redirect to ensure proper default document resolution (index.html):
```caddy
# [Description]
redir /path /path/ 308
handle_path /path/* {
    root * absolute_directory_path
    file_server
}
```

## 4. Key AI Pair-Programming Pitfalls Solved
Future agents must preserve these solutions:
1. **Subprocess Deadlocks**: When starting daemon-like background processes (like Caddy or cloudflared) in Python, do NOT redirect `stdout`/`stderr` to `subprocess.PIPE` if you call `.communicate()` or wait. Instead, pipe them to `subprocess.DEVNULL` or open logs using file descriptors (e.g., `open("cloudflared.log", "a")`) to prevent buffers from filling up and blocking Python's execution threads.
2. **Caddy Logging Trailing Slash Nuances**: Caddy's global `log` block defines formatting, but for site-level access logging, the `log` block must be defined **inside the site block** in the Caddyfile. Furthermore, access loggers in Caddy use names like `http.log.access.log0` instead of exactly `http.log.access`. Use `startswith("http.log.access")` when parsing log lines.
3. **Cloudflare Tunnel Ingress Config**: Remotely managed tunnels require a `PUT /accounts/{account_id}/cfd_tunnel/{tunnel_id}/configurations` call to update ingress configurations. Always append a catch-all rules fallback at the end: `{"service": "http_status:404"}`.
