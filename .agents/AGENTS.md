# FALO Project Custom Rules

## 1. 文件交付規範 (Documentation Deliverables Rule)
本專案（及後續所有關聯專案）在建立任何技術研究、架構規劃或操作 SOP 時，必須遵循以下「三合一」文件格式規範：

1. **Markdown 格式 (.md)**：
   - 定位：**AI 讀取與處理**。
   - 要求：結構嚴謹、使用標準 GFM (GitHub Flavored Markdown) 格式、清晰的程式碼區塊、正確的絕對/相對連結。
2. **HTML 格式 (.html)**：
   - 定位：**人類閱讀 (Human-friendly)**。
   - 要求：具備高品質的美學設計（包含漸層、現代字型如 Inter/Outfit、乾淨的表格、響應式排版），便於網頁瀏覽。
3. **HTML 教材檔 (.html)**：
   - 定位：**AI 老師 / 顧問的培訓與簡報教材**。
   - 要求：採用 Slides 投影片風格（如基於 Reveal.js 的單頁投影片結構或大型區塊），著重於痛點剖析、架構對比、流程圖示，便於現場教學或顧問說明。
